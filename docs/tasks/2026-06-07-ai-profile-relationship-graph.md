# 任务：AI 人物画像关系图谱

## 背景

用户希望把 `20260606-广州大课现场群-生财通讯录整理.xlsx` 中大量成员自我介绍做成关系图谱，让成员因为业务、资源、需求和合作可能性被匹配在一起。

第一版已完成规则型关系图谱页面，随后用户确认继续优化：

- 第二版：AI 抽取结构化人物画像
- 第三版：生成高质量 person-person 关系和理由
- 第四版：优化图谱交互和详情抽屉
- 第五版：做首页正式入口和演示流程
- 兼顾移动端页面展示

## 目标

1. 批量调用 AI，把 matched 成员介绍抽取为结构化人物画像。
2. 基于画像生成高质量人物关系，包含关系类型、分数、理由、证据和破冰话术。
3. 新增或升级关系图谱页面，用于展示人物、业务标签、城市和人物间关系。
4. 让 `/api/graph` 优先读取 AI 生成图谱数据。
5. 兼顾桌面端和移动端展示效果。

## 任务类型

新功能 / 数据生成 / 前端交互优化 / API 增强

## AI 执行记录

- 阅读了：
  - `任务记录规则_空白模板.md`
  - `files/server.mjs`
  - `files/graph.html`
  - 通讯录整理 JSON 数据结构
- 检查了：
  - 当前 Git 状态
  - `/api/graph` 返回结构
  - AI API 配置的 base URL 和 model，未打印完整密钥
  - Playwright 桌面端和移动端截图
- 发现问题：
  - 当前 OpenAI 兼容服务的 `/v1/responses` 会返回空 `output`。
  - 批量 8 人请求时偶发不可解析 JSON。
  - AI 画像生成的细粒度标签过多，直接进入画布会导致图谱过密。
- 采用方案：
  - 批处理脚本优先使用 `/v1/chat/completions`，并保留 `/v1/responses` 兜底。
  - 批量大小调整为 4 人一批，提高 AI 返回稳定性。
  - 图谱画布只展示出现频次较高的 Top 100 标签，完整画像保留在详情面板。
  - `/api/graph` 有 AI 图谱数据时使用 `ai-profile-v2`，否则回退第一版规则图谱。

## 遗留问题

1. 关系生成目前是“AI 画像 + 本地算法匹配”，不是每条关系再调用 AI 精修。
2. 少数画像可能仍需要人工校正行业词或资源词。
3. 后续可以增加“只看某个人的一度/二度关系”模式，进一步降低图谱密度。
