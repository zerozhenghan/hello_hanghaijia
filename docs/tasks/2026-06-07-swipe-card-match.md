# 任务：探探式滑卡人脉匹配体验

## 背景

当前网站已经支持基于现场群整理结果进行 AI 批次推荐：用户输入诉求后，每批以列表形式展示最多 4 张人脉卡片，并支持“换一批”“查看上一批”和历史排重。

用户希望把体验从“搜索结果页”升级为更有产品感的“人脉雷达 / AI 牵线官”：AI 一次推荐 8 人，前端以探探式单张滑卡呈现，用户左滑跳过、右滑加入“想认识”，滑完后展示精选联系人列表和破冰话术。

## 目标

1. 先完成 Git 安全管理，确保不满意时能回撤到上一个版本。
2. 新建本任务记录。
3. 后端一次返回 8 位推荐人，继续沿用已有 `excludeIds` 排重机制。
4. 前端从列表式 4 张卡片改为单张滑卡队列。
5. 支持：
   - 右滑 / 按钮：想认识
   - 左滑 / 按钮：跳过
   - 详情展开：查看推荐理由、契合点、破冰话术
6. 滑完一批后展示：
   - 已选中的人
   - 每个人的破冰话术
   - 一键复制话术
   - 再来一批，并排除历史所有推荐过的人

## 任务类型

中 / 大改动：前端主交互改造 + 后端推荐数量调整 + Git 安全点管理。
